---
name: show-me
description: Use when user is unhappy with the UI/UX, wants to compare layouts, redesign a component, or make any visual frontend decision. Also triggers on "show me", "don't like the UX", "wireframe", "mockup" or "I don't like how this looks".
dependencies:
  - plugin: superpowers:brainstorming
---

Don't describe layouts in text. Show me in the browser.

Read and follow the visual companion guide from superpowers:brainstorming to start the server and show mockups. If the server can't start, fall back to rendering a standalone HTML file and opening it in the browser. When comparing approaches, build each option and let me pick in the browser. Once I pick a direction, confirm the choice and move on.
